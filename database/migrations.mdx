## Overview

This page contains the complete SQL migration script to create the Toluca Altas Montañas database schema in Supabase/PostgreSQL.

<Warning>
  The migration script includes `DROP TABLE` statements that will **delete all existing data**. Only run this on a new database or when you explicitly want to reset the schema.
</Warning>

## Running the Migration

### Using Supabase Dashboard

1. Navigate to your Supabase project
2. Go to **SQL Editor** in the left sidebar
3. Click **New Query**
4. Copy and paste the migration script below
5. Click **Run** to execute

### Using psql

```bash
psql "postgresql://postgres:[YOUR-PASSWORD]@db.[YOUR-PROJECT].supabase.co:5432/postgres" < migration.sql
```

## Complete Migration Script

```sql
begin;

-- 0) DROP EXISTING TABLES (WARNING: Deletes all data)
drop table if exists public.player_registrations cascade;
drop table if exists public.schedules cascade;
drop table if exists public.categories cascade;
drop table if exists public.venues cascade;

-- 1) FUNCTION + TRIGGERS for updated_at
create or replace function public.set_updated_at()
returns trigger
language plpgsql
as $$
begin
new.updated_at = now();
return new;
end;
$$;

-- 2) SEDES (Venues)
create table public.venues (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  place text not null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint venues_unique unique (name, place)
);

create trigger venues_set_updated_at
before update on public.venues
for each row execute function public.set_updated_at();

-- 3) CATEGORÍAS (Categories)
create table public.categories (
  id uuid primary key default gen_random_uuid(),
  name text not null unique,
  year_from int not null,
  year_to int not null,
  sort_order int not null default 0,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint categories_year_range check (year_from <= year_to),
  constraint categories_year_limits check (
    year_from between 1900 and (extract(year from now())::int + 1)
    and year_to between 1900 and (extract(year from now())::int + 1)
  )
);

create trigger categories_set_updated_at
before update on public.categories
for each row execute function public.set_updated_at();

-- 4) HORARIOS (Schedules)
create table public.schedules (
  id uuid primary key default gen_random_uuid(),
  category_id uuid not null references public.categories(id) on delete cascade,
  venue_id uuid not null references public.venues(id) on delete cascade,
  weekday smallint not null, -- 1=Monday ... 7=Sunday
  start_time time not null,
  end_time time not null,
  is_optional boolean not null default false,
  note text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint schedules_weekday_valid check (weekday between 1 and 7),
  constraint schedules_time_valid check (start_time < end_time),
  constraint schedules_unique unique (category_id, venue_id, weekday, start_time, end_time)
);

create trigger schedules_set_updated_at
before update on public.schedules
for each row execute function public.set_updated_at();

create index schedules_by_category on public.schedules(category_id);
create index schedules_by_venue on public.schedules(venue_id);

-- 5) REGISTROS (Player Registrations)
create table public.player_registrations (
  id uuid primary key default gen_random_uuid(),
  full_name text not null,
  birth_date date not null,
  birth_year smallint generated always as ((extract(year from birth_date))::smallint) stored,
  phone text not null,
  venue_id uuid not null references public.venues(id) on delete restrict,
  category_id uuid not null references public.categories(id) on delete restrict,
  created_at timestamptz not null default now()
);

create index if not exists idx_player_reg_birth_year on public.player_registrations(birth_year);
create index if not exists idx_player_reg_venue on public.player_registrations(venue_id);
create index if not exists idx_player_reg_category on public.player_registrations(category_id);

commit;
```

## Migration Breakdown

### Step 0: Drop Existing Tables

```sql
drop table if exists public.player_registrations cascade;
drop table if exists public.schedules cascade;
drop table if exists public.categories cascade;
drop table if exists public.venues cascade;
```

Drops tables in dependency order to avoid foreign key constraint violations:
1. `player_registrations` (depends on venues and categories)
2. `schedules` (depends on venues and categories)
3. `categories` (no dependencies)
4. `venues` (no dependencies)

<Note>
  The `CASCADE` option ensures that dependent objects (like triggers and indexes) are also dropped.
</Note>

---

### Step 1: Create Timestamp Trigger Function

```sql
create or replace function public.set_updated_at()
returns trigger
language plpgsql
as $$
begin
new.updated_at = now();
return new;
end;
$$;
```

This reusable trigger function automatically updates the `updated_at` timestamp whenever a row is modified.

**Usage**: Attached to tables that need automatic timestamp tracking.

---

### Step 2: Create venues Table

```sql
create table public.venues (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  place text not null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint venues_unique unique (name, place)
);

create trigger venues_set_updated_at
before update on public.venues
for each row execute function public.set_updated_at();
```

**Key Features**:
- UUID primary key with automatic generation
- Unique constraint on `(name, place)` combination
- Automatic timestamp tracking via trigger

---

### Step 3: Create categories Table

```sql
create table public.categories (
  id uuid primary key default gen_random_uuid(),
  name text not null unique,
  year_from int not null,
  year_to int not null,
  sort_order int not null default 0,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint categories_year_range check (year_from <= year_to),
  constraint categories_year_limits check (
    year_from between 1900 and (extract(year from now())::int + 1)
    and year_to between 1900 and (extract(year from now())::int + 1)
  )
);
```

**Key Features**:
- Unique category names
- Validation that `year_from <= year_to`
- Year range validation (1900 to current year + 1)
- Sort order for UI display

---

### Step 4: Create schedules Table

```sql
create table public.schedules (
  id uuid primary key default gen_random_uuid(),
  category_id uuid not null references public.categories(id) on delete cascade,
  venue_id uuid not null references public.venues(id) on delete cascade,
  weekday smallint not null,
  start_time time not null,
  end_time time not null,
  is_optional boolean not null default false,
  note text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint schedules_weekday_valid check (weekday between 1 and 7),
  constraint schedules_time_valid check (start_time < end_time),
  constraint schedules_unique unique (category_id, venue_id, weekday, start_time, end_time)
);
```

**Key Features**:
- Foreign keys to venues and categories with CASCADE delete
- Weekday validation (1-7)
- Time validation (start < end)
- Unique constraint prevents duplicate schedules
- Indexes on foreign keys for performance

---

### Step 5: Create player_registrations Table

```sql
create table public.player_registrations (
  id uuid primary key default gen_random_uuid(),
  full_name text not null,
  birth_date date not null,
  birth_year smallint generated always as ((extract(year from birth_date))::smallint) stored,
  phone text not null,
  venue_id uuid not null references public.venues(id) on delete restrict,
  category_id uuid not null references public.categories(id) on delete restrict,
  created_at timestamptz not null default now()
);
```

**Key Features**:
- **Generated column**: `birth_year` automatically computed from `birth_date`
- Foreign keys with RESTRICT delete (prevents deleting venues/categories in use)
- Indexes on foreign keys and birth_year for query performance
- No `updated_at` field (registrations are not typically modified)

<Warning>
  Never manually insert or update the `birth_year` column. PostgreSQL will raise an error if you try.
</Warning>

## Transaction Safety

The migration is wrapped in a transaction:

```sql
begin;
-- ... migration statements ...
commit;
```

**Benefits**:
- **All-or-nothing**: Either all changes succeed or all are rolled back
- **Consistency**: Database never in a half-migrated state
- **Safety**: If any error occurs, no changes are applied

## Verification Queries

After running the migration, verify the schema:

### Check Tables Exist

```sql
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public'
ORDER BY table_name;
```

### Check Foreign Keys

```sql
SELECT
  tc.table_name,
  kcu.column_name,
  ccu.table_name AS foreign_table_name,
  ccu.column_name AS foreign_column_name,
  rc.delete_rule
FROM information_schema.table_constraints AS tc
JOIN information_schema.key_column_usage AS kcu
  ON tc.constraint_name = kcu.constraint_name
JOIN information_schema.constraint_column_usage AS ccu
  ON ccu.constraint_name = tc.constraint_name
JOIN information_schema.referential_constraints AS rc
  ON rc.constraint_name = tc.constraint_name
WHERE tc.constraint_type = 'FOREIGN KEY'
  AND tc.table_schema = 'public'
ORDER BY tc.table_name;
```

### Check Indexes

```sql
SELECT
  tablename,
  indexname,
  indexdef
FROM pg_indexes
WHERE schemaname = 'public'
ORDER BY tablename, indexname;
```

## Rollback

If you need to remove the schema completely:

```sql
begin;

drop table if exists public.player_registrations cascade;
drop table if exists public.schedules cascade;
drop table if exists public.categories cascade;
drop table if exists public.venues cascade;
drop function if exists public.set_updated_at() cascade;

commit;
```
