## Overview

The Registrations section is the primary interface for administrators to view and manage all player registration data collected through the landing page. It provides a comprehensive table view with filtering, searching, and export capabilities.

## Data Loading

Registrations are loaded from Supabase with related venue and category information:

```typescript
const load = async () => {
  setLoading(true);
  setError(null);

  const supabase = createClient();

  const { data, error } = await supabase
    .from("player_registrations")
    .select(
      `
        id,
        full_name,
        birth_date,
        phone,
        created_at,
        venue:venues ( id, name, place ),
        category:categories ( id, name, year_from, year_to, sort_order )
      `
    )
    .order("created_at", { ascending: false });

  if (error) {
    setError(error.message);
    setRows([]);
  } else {
    const mapped: RegistrationRow[] = (data ?? []).map((r: any) => ({
      id: r.id,
      full_name: r.full_name,
      birth_date: r.birth_date,
      phone: r.phone,
      created_at: r.created_at,
      venue: r.venue ?? null,
      category: r.category ?? null,
    }));
    setRows(mapped);
  }

  setLoading(false);
};
```

<Note>
The query uses Supabase's relation syntax to join `venues` and `categories` tables, providing complete registration information in a single request.
</Note>

## Registration Table

### Table Columns

The registration table displays the following information:

| Column | Description | Source |
|--------|-------------|--------|
| **Jugador** | Player's full name and registration date | `full_name`, `created_at` |
| **Edad** | Calculated age based on birth date | Calculated from `birth_date` |
| **Teléfono** | Contact phone number | `phone` |
| **Categoría** | Age category assignment | `categories.name` |
| **Sede** | Preferred training venue | `venues.name` |
| **Acciones** | Copy phone and view details buttons | - |

### Age Calculation

Player age is dynamically calculated based on birth date:

```typescript
function calcAge(birthDate: string) {
  const d = new Date(birthDate);
  if (Number.isNaN(d.getTime())) return null;
  const now = new Date();
  let age = now.getFullYear() - d.getFullYear();
  const m = now.getMonth() - d.getMonth();
  if (m < 0 || (m === 0 && now.getDate() < d.getDate())) age--;
  return age;
}
```

This accounts for whether the player's birthday has occurred this year.

### Date Formatting

Registration dates are formatted in Spanish locale:

```typescript
function formatFecha(s: string) {
  const d = new Date(s);
  if (Number.isNaN(d.getTime())) return "—";
  return new Intl.DateTimeFormat("es-MX", {
    day: "2-digit",
    month: "short",
    year: "numeric",
  }).format(d);
}
```

Example output: `14 mar 2026`

## Handling Missing Data

The system gracefully handles missing or null data:

```typescript
function normalize(v: any, fallback: string) {
  return v == null || v === "" ? fallback : String(v);
}

// Usage
const sede = normalize(r.venue?.name, "Sin sede");
const cat = normalize(r.category?.name, "Sin categoría");
```

This ensures:
- Missing venues display as "Sin sede"
- Missing categories display as "Sin categoría"
- Missing ages display as "—"

If data loading fails, an error message is displayed:

```typescript
{error ? (
  <div className="rounded-xl border border-[#D50032]/25 bg-[#D50032]/10 p-4">
```

<Warning>

- Database connection issues
- Permission errors (RLS policies)
- Network timeouts

Check your Supabase connection and Row Level Security policies if you encounter errors.
