## Overview

The admin dashboard includes a powerful export feature that allows you to download registration data as CSV (Comma-Separated Values) files. This enables offline analysis, backup, and integration with other tools like Excel, Google Sheets, or data analysis software.

## Quick Start

<Steps>
  <Step title="Apply filters (optional)">
    Use search and filters to narrow down the registrations you want to export.
  </Step>
  
  <Step title="Click Exportar button">
    Click the "Exportar" button in the top toolbar.
  </Step>
  
  <Step title="Download CSV file">
    Your browser will automatically download a CSV file with the filtered results.
  </Step>
</Steps>

<Note>
The export includes **only the currently filtered registrations**, not all registrations in the database. Always verify your active filters before exporting.
</Note>

## Export Implementation

### Export Function

The export functionality is implemented as a client-side CSV generator:

```typescript
function exportCSV(rows: RegistrationRow[]) {
  const headers = ["Nombre", "Edad", "Teléfono", "Sede", "Categoría", "Fecha"];

  const lines = rows.map((r) => {
    const sede = normalize(r.venue?.name, "Sin sede");
    const cat = normalize(r.category?.name, "Sin categoría");
    const age = calcAge(r.birth_date);
    const fecha = formatFecha(r.created_at);

    const cells = [
      r.full_name ?? "",
      age == null ? "" : String(age),
      r.phone ?? "",
      sede,
      cat,
      fecha,
    ].map((x) => `"${String(x).replaceAll('"', '""')}"`);

    return cells.join(",");
  });

  const csv = [headers.join(","), ...lines].join("\n");
  const blob = new Blob([csv], { type: "text/csv;charset=utf-8;" });
  const url = URL.createObjectURL(blob);

  const a = document.createElement("a");
  a.href = url;
  a.download = `registros_${new Date().toISOString().slice(0, 10)}.csv`;
  document.body.appendChild(a);
  a.click();
  a.remove();
  URL.revokeObjectURL(url);
}
```

## CSV Structure

### Column Headers

The exported CSV includes six columns:

| Column Name | Description | Data Source |
|-------------|-------------|-------------|
| **Nombre** | Player's full name | `full_name` |
| **Edad** | Current age in years | Calculated from `birth_date` |
| **Teléfono** | Contact phone number | `phone` |
| **Sede** | Preferred training venue | `venues.name` |
| **Categoría** | Age category | `categories.name` |
| **Fecha** | Registration date | `created_at` (formatted) |

## Data Processing

### Age Calculation

Player age is calculated at export time based on current date:

```typescript
function calcAge(birthDate: string) {
  const d = new Date(birthDate);
  if (Number.isNaN(d.getTime())) return null;
  const now = new Date();
  let age = now.getFullYear() - d.getFullYear();
  const m = now.getMonth() - d.getMonth();
  if (m < 0 || (m === 0 && now.getDate() < d.getDate())) age--;
  return age;
}
```

This ensures exported ages are accurate as of the export date.

### Date Formatting

Registration dates are formatted in Spanish locale:

```typescript
function formatFecha(s: string) {
  const d = new Date(s);
  if (Number.isNaN(d.getTime())) return "—";
  return new Intl.DateTimeFormat("es-MX", {
    day: "2-digit",
    month: "short",
    year: "numeric",
  }).format(d);
}
```

Output format: `DD MMM YYYY` (e.g., "14 mar 2026")

## CSV Encoding

### Character Encoding

The CSV is exported with UTF-8 encoding to properly support Spanish characters:

```typescript
const blob = new Blob([csv], { type: "text/csv;charset=utf-8;" });
```

This ensures proper display of:
- Accented characters (á, é, í, ó, ú)
- Special characters (ñ, ¿, ¡)
- Proper Spanish formatting

### CSV Escaping

Field values are properly escaped according to CSV standards:

```typescript
const cells = [
  r.full_name ?? "",
  age == null ? "" : String(age),
  r.phone ?? "",
  sede,
  cat,
  fecha,
].map((x) => `"${String(x).replaceAll('"', '""')}"`);
```

This handles:
- Commas in names (e.g., "García, Juan")
- Quotes in text (doubled: "" instead of ")
- Line breaks in fields

<Note>
All fields are quoted to prevent issues with special characters. This is the safest approach for CSV generation.
</Note>

## File Naming

### Automatic Filename

Exported files are automatically named with the current date:

```typescript
a.download = `registros_${new Date().toISOString().slice(0, 10)}.csv`;
```

Filename format: `registros_YYYY-MM-DD.csv`

Examples:
- `registros_2026-03-14.csv`
- `registros_2026-12-25.csv`

<Tip>
The ISO date format (YYYY-MM-DD) sorts chronologically when viewed in file explorers, making it easy to organize exports by date.
</Tip>

## Export Limitations

<Warning>
**Current Limitations:**

1. **Client-side only**: Export happens in the browser. Very large datasets (>10,000 rows) may cause performance issues.

2. **No server-side backup**: Exports are not automatically backed up. Save important exports to a secure location.

3. **Fixed columns**: You cannot customize which columns to include in the export.

4. **No format options**: Export is always CSV. PDF, Excel, or JSON formats are not available.

5. **No scheduled exports**: Exports must be triggered manually. No automatic/scheduled exports.
</Warning>

## Troubleshooting

### Export Not Downloading

**Solution**: Check browser permissions
- Ensure pop-ups are not blocked
- Check browser download settings
- Try a different browser

### Special Characters Not Displaying

**Solution**: Check file encoding
- Open CSV in a UTF-8 compatible editor
- In Excel, use "Get Data" > "From Text/CSV" and specify UTF-8 encoding

### Commas Breaking Columns

**Solution**: Fields should be quoted
- This is handled automatically by the export function
- If you see issues, report it as a bug—all fields should be quoted

### Empty Export File

**Solution**: Check active filters
- Result counter shows "0"? No records match your filters
- Clear filters and try again
