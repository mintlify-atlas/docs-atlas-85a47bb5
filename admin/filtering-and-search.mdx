## Overview

The admin dashboard provides robust filtering and search functionality to help you quickly locate specific registrations. All filtering happens client-side for instant results without additional database queries.

## Search Bar

### Full-Text Search

The search bar performs real-time full-text search across multiple fields:

```typescript
const [q, setQ] = useState("");

// Search implementation
if (q.trim()) {
  const needle = q.trim().toLowerCase();
  filtrados = filtrados.filter((r) => {
    const nombre = String(r.full_name ?? "").toLowerCase();
    const tel = String(r.phone ?? "").toLowerCase();
    const sede = String(r.venue?.name ?? "").toLowerCase();
    const cat = String(r.category?.name ?? "").toLowerCase();
    return (
      nombre.includes(needle) ||
      tel.includes(needle) ||
      sede.includes(needle) ||
      cat.includes(needle)
    );
  });
}
```

## Venue Filter

### Filter by Venue

The venue dropdown allows you to filter registrations by their preferred training location:

```typescript
const [sedeSel, setSedeSel] = useState("Todas");

// Extract unique venues from data
const sedesSet = new Set<string>();
for (const r of safe) {
  sedesSet.add(normalize(r.venue?.name, "Sin sede"));
}
const sedes = ["Todas", ...Array.from(sedesSet).sort()];

// Apply venue filter
if (sedeSel !== "Todas") {
  filtrados = filtrados.filter(
    (r) => normalize(r.venue?.name, "Sin sede") === sedeSel
  );
}
```

### Venue Filter Options

The dropdown automatically populates with:
- **Todas**: Shows all registrations (default)
- All unique venue names from existing registrations (alphabetically sorted)
- **Sin sede**: Registrations without an assigned venue

```typescript
for (const r of safe) {

const categorias = ["Todas", ...Array.from(catsSet).sort()];

if (catSel !== "Todas") {
  filtrados = filtrados.filter(
    (r) => normalize(r.category?.name, "Sin categoría") === catSel
  );

```

### Category Filter Options

The dropdown includes:
- **Todas**: Shows all registrations (default)
- All unique category names (alphabetically sorted)
- **Sin categoría**: Registrations without a category assignment

<Note>
Categories are typically age-based (e.g., Sub-13, Sub-15, Sub-17) and are automatically assigned based on the player's birth date during registration.
</Note>

### Filter Composition

Filters work together in a logical AND relationship:

```typescript

    filtrados = filtrados.filter(
```

## Filter Performance

### Memoization

All filter computations are memoized to prevent unnecessary recalculation:

```typescript
const data = useMemo(() => {
  // All filtering logic...
}, [rows, q, sedeSel, catSel]);
```

This means:
- Filters only recalculate when `rows`, `q`, `sedeSel`, or `catSel` change
- Re-renders don't cause filter re-computation
- Provides instant filtering even with thousands of registrations

## Filter State Persistence

### Current Behavior

Filter state is stored in React component state:

```typescript
const [q, setQ] = useState("");
const [sedeSel, setSedeSel] = useState("Todas");
const [catSel, setCatSel] = useState("Todas");
```

<Note>
**Important**: Filter state is **not persisted** across page reloads or navigation. When you refresh the page or navigate away and back, all filters reset to their defaults.
</Note>

### Potential Enhancement

Consider implementing URL-based filter persistence:

```typescript
// Example: Store filters in URL query params
const router = useRouter();
const searchParams = useSearchParams();

// Read filters from URL
const q = searchParams.get('search') ?? '';
const sedeSel = searchParams.get('venue') ?? 'Todas';
const catSel = searchParams.get('category') ?? 'Todas';

// Update URL when filters change
const updateFilters = (newFilters) => {
  const params = new URLSearchParams();
  if (newFilters.search) params.set('search', newFilters.search);
  if (newFilters.venue !== 'Todas') params.set('venue', newFilters.venue);
  if (newFilters.category !== 'Todas') params.set('category', newFilters.category);
  router.push(`?${params.toString()}`);
};
```

This would allow:
- Sharable filtered views via URL
- Browser back/forward navigation through filter states
- Filter persistence across page reloads
