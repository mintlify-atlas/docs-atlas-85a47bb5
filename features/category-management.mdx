## Overview

The Category Management system allows administrators to define age-based categories that automatically determine which group a player belongs to based on their birth year. Categories are central to the registration and scheduling systems.

<Note>
  Categories use year ranges (year_from and year_to) to automatically assign players during registration. The system finds the matching category based on the player's birth year.
</Note>

## Data Model

### Category Type

```typescript
type Category = {
  id: string;
  name: string;
  year_from: number;
  year_to: number;
  sort_order: number | null;
};
```

### Database Table: `categories`

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `name` | TEXT | Category name (e.g., "Sub-10", "Sub-12") |
| `year_from` | INTEGER | Starting birth year for the category |
| `year_to` | INTEGER | Ending birth year for the category |
| `sort_order` | INTEGER | Display order (null allowed) |

## Category Assignment Logic

The automatic category assignment uses a range-matching algorithm:

```typescript components/player-registration-form.tsx
function findCategory(birthYear: number, cats: Category[]) {
  return cats.find((c) => {
    const min = Math.min(c.year_from, c.year_to);
    const max = Math.max(c.year_from, c.year_to);
    return birthYear >= min && birthYear <= max;
  });
}
```

### How It Works

<Steps>
  <Step title="Player Enters Birth Date">

  </Step>
  <Step title="Extract Birth Year">

  <Step title="Find Matching Category">
    System searches for a category where birth year falls within the range

## Data Loading in Components

### Player Registration Form

Categories are loaded on mount for immediate category detection:

```tsx components/player-registration-form.tsx
const [cats, setCats] = React.useState<Category[]>([]);

React.useEffect(() => {
  const run = async () => {
    const supabase = createClient();

    const cRes = await supabase
      .from("categories")
      .select("id, name, year_from, year_to, sort_order")
      .order("sort_order", { ascending: true });

    if (cRes.error) {
      setError(cRes.error.message);
    } else {
      setCats((cRes.data ?? []) as Category[]);
    }
  };

  run();
}, []);
```

### Admin Dashboard

## Sort Order

The `sort_order` field controls display sequence:

- Lower numbers appear first
- Null values appear last
- Used in dropdowns and grouped displays

```typescript
const entries = Array.from(map.entries()).sort((a, b) => {
  const sa = catById.get(a[0])?.sort_order ?? 9999;
  const sb = catById.get(b[0])?.sort_order ?? 9999;
  return sa - sb;
});
```

## Year Range Validation

<Warning>
  Ensure year ranges don't overlap between categories to avoid ambiguous assignments.
</Warning>

### Good Practice

```json
[
  {"name": "Sub-10", "year_from": 2016, "year_to": 2017},
  {"name": "Sub-12", "year_from": 2014, "year_to": 2015}
]
```

No overlap - clear boundaries between categories.

### Bad Practice

```json
[
  {"name": "Sub-10", "year_from": 2015, "year_to": 2017},
  {"name": "Sub-12", "year_from": 2014, "year_to": 2016}
]
```

Years 2015-2016 overlap - ambiguous assignment.

## Handling Missing Categories

If no category matches the birth year:

```tsx components/player-registration-form.tsx
if (!category) return setError("No se encontró categoría para esa fecha.");
```

The form displays "No encontrada" and prevents submission.

## Database Relationships

Categories have foreign key relationships with:

<CardGroup cols={2}>
  <Card title="player_registrations" icon="user">
    Each registration references a category
  </Card>
  <Card title="schedules" icon="calendar">
    Each schedule is assigned to a category
  </Card>
</CardGroup>

### Example Query with Relationships

```sql
SELECT 
  c.name as category_name,
  COUNT(DISTINCT pr.id) as player_count,
  COUNT(DISTINCT s.id) as schedule_count
FROM categories c
LEFT JOIN player_registrations pr ON pr.category_id = c.id
LEFT JOIN schedules s ON s.category_id = c.id
GROUP BY c.id, c.name, c.sort_order
ORDER BY c.sort_order;
```
